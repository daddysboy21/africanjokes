# Contributing to africanjokes

Thanks for wanting to contribute. The most welcome contributions are **new
jokes and proverbs** from across Africa — especially from countries or
language groups not yet well represented.

This repository ships **two packages** that share the same data:

- `africanjokes` on PyPI (Python, in [`africanjokes/`](africanjokes/))
- `africanjokes` on npm (JavaScript, in [`npm/`](npm/))

The canonical data files live in
[`africanjokes/data/`](africanjokes/data/). The npm package keeps a copy in
[`npm/src/data/`](npm/src/data/) that is regenerated by a sync script — you
edit one place.

---

## Adding jokes

1. Open [`africanjokes/data/jokes.json`](africanjokes/data/jokes.json).
2. Add an entry to the `jokes` array:

   ```json
   {
     "text": "Your joke text here.",
     "country": "Nigeria",
     "themes": ["food", "family"]
   }
   ```

3. Rules:
   - `text` is the joke as it should be displayed (any Unicode is fine — the
     CLI handles UTF-8 on Windows consoles too).
   - `country` is one of the recognised values, or `"Pan-African"` if not
     country-specific. New countries are welcome — add them organically.
   - `themes` is **a non-empty list** drawn from the controlled vocabulary
     defined in [`tools/validate_data.py`](tools/validate_data.py)
     (`ALLOWED_THEMES`). If your joke really needs a new theme, add it
     to that set in the same PR.

4. Run the validator and sync the npm copy:

   ```bash
   python tools/validate_data.py
   node tools/sync_data.mjs
   ```

5. Commit both files (`africanjokes/data/jokes.json` and
   `npm/src/data/jokes.json`).

## Adding proverbs

1. Open [`africanjokes/data/proverbs.json`](africanjokes/data/proverbs.json).
2. Add an entry to the `proverbs` array:

   ```json
   {
     "text": "When the spider webs unite, they can tie up a lion.",
     "country": "Ethiopia",
     "attribution": "Ethiopian"
   }
   ```

3. Rules:
   - **Authenticity matters here.** Please add only proverbs you can verify
     are traditionally attested in the named culture / language. Don't
     invent proverbs.
   - `attribution` should name the people or language group (e.g.
     `"Yoruba"`, `"Akan"`, `"Swahili"`, `"Wolof"`). If you only know the
     country, use the demonym (e.g. `"Liberian"`).
   - `country` is the modern country most associated with that people /
     language group.

4. Re-run the validator and sync (same as above).

## Code changes

### Python package

```bash
pip install -e ".[test]"
pytest
python tools/validate_data.py
```

### npm package

```bash
cd npm
npm test                       # runs node:test on the test/ directory
node ../tools/sync_data.mjs    # if you edited data
```

---

## Pull request checklist

- [ ] Data validator passes: `python tools/validate_data.py`
- [ ] npm copy is in sync: `node tools/sync_data.mjs`
- [ ] Python tests pass: `pytest`
- [ ] Node tests pass: `cd npm && npm test`
- [ ] If you bumped behavior, [`CHANGELOG.md`](CHANGELOG.md) is updated
      under `[Unreleased]`.
- [ ] If you added a new theme, it is also added to `ALLOWED_THEMES` in
      `tools/validate_data.py`.

---

## Releasing (maintainer only)

1. Move `[Unreleased]` notes in `CHANGELOG.md` under a new dated version.
2. Bump the version in **all three** places:
   `pyproject.toml`, `africanjokes/__init__.py` (`__version__`), and
   `npm/package.json`.
3. Re-run the validator, sync, and both test suites.
4. PyPI: `python -m build && twine upload dist/*`.
5. npm: `cd npm && npm publish` (the `prepack` script re-runs the sync).
6. Tag the release: `git tag v1.x.y && git push --tags`.

---

## Code of conduct

Be kind, be specific, be self-aware. Keep humor warm — this library is
about celebrating, not punching down. Jokes that lean on harmful
stereotypes will be declined.
